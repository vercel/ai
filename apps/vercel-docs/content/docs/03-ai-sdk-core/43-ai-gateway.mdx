# Use model providers through AI Gateway

The [Vercel AI Gateway](/providers/ai-sdk-providers/ai-gateway) is the recommended way to connect to model providers when using the AI SDK. Instead of installing and configuring separate provider packages for OpenAI, Anthropic, Google, and others, you access all of them through a single interface with built-in observability, automatic authentication on Vercel, and no extra dependencies.

## Prerequisites

- A Vercel account.
- An AI Gateway API key. You can create one from the [Vercel dashboard](https://vercel.com/d?to=%2F%5Bteam%5D%2F%7E%2Fai-gateway%2Fapi-keys).
- The `ai` package installed in your project.

## Why use AI Gateway

When you use dedicated provider packages, you install a separate dependency for each provider, manage individual API keys, and handle provider-specific configuration. AI Gateway removes this overhead:

- **Single dependency**: The `ai` package includes the AI Gateway provider. No additional packages to install.
- **Unified authentication**: One API key or automatic OIDC authentication on Vercel deployments.
- **Switch providers without code changes**: Change the model string from `openai/gpt-5` to `anthropic/claude-sonnet-4.5` and your code works the same way.
- **Built-in observability**: Monitor model usage, costs, and performance from the Vercel dashboard.
- **Automatic OIDC on Vercel**: When deployed to Vercel, authentication is handled automatically — no API key management needed in production.

## Set up your API key

Add your AI Gateway API key to your environment:

```env filename=".env.local"
AI_GATEWAY_API_KEY=your_api_key_here
```

When deployed to Vercel, you can use [OIDC authentication](/providers/ai-sdk-providers/ai-gateway#oidc-authentication-vercel-deployments) instead, which handles tokens automatically without an API key.

## Generate text with AI Gateway

AI Gateway is the default global provider in the AI SDK. Pass a model string in the `provider/model-name` format and the SDK routes the request through AI Gateway:

```ts
import { generateText } from 'ai';

const { text } = await generateText({
  model: 'openai/gpt-5',
  prompt: 'Explain how a transformer neural network works.',
});

console.log(text);
```

To switch to a different provider, change the model string:

```ts
import { generateText } from 'ai';

const { text } = await generateText({
  model: 'anthropic/claude-sonnet-4.5',
  prompt: 'Explain how a transformer neural network works.',
});

console.log(text);
```

No configuration changes, no new dependencies, and no new API keys.

## Stream text with AI Gateway

For interactive use cases like chatbots, use `streamText` to receive tokens as they're generated:

```ts
import { streamText } from 'ai';

const result = streamText({
  model: 'google/gemini-2.5-flash',
  prompt: 'Write a short guide on getting started with TypeScript.',
});

for await (const textPart of result.textStream) {
  process.stdout.write(textPart);
}
```

## Use the gateway provider instance

You can also import the `gateway` provider instance for explicit usage. This is useful when you need to pass a provider to functions that expect a provider object, or when working with [middleware](/docs/ai-sdk-core/middleware) and [provider management](/docs/ai-sdk-core/provider-management):

```ts
import { generateText, gateway } from 'ai';

const { text } = await generateText({
  model: gateway('anthropic/claude-sonnet-4.5'),
  prompt: 'What are the benefits of edge computing?',
});

console.log(text);
```

## Create a custom gateway instance

If you need to customize the gateway configuration — for example, to set a specific API key or add custom headers — use `createGateway`:

```ts
import { createGateway, generateText } from 'ai';

const myGateway = createGateway({
  apiKey: process.env.AI_GATEWAY_API_KEY ?? '',
});

const { text } = await generateText({
  model: myGateway('openai/gpt-5'),
  prompt: 'Summarize the benefits of serverless computing.',
});

console.log(text);
```

## Use AI Gateway in a Next.js API route

Here's a complete example of using AI Gateway in a Next.js App Router API route with streaming:

```ts filename="app/api/chat/route.ts"
import { streamText } from 'ai';

export const maxDuration = 60;

export async function POST(request: Request) {
  const { prompt }: { prompt?: string } = await request.json();

  if (!prompt) {
    return new Response('Prompt is required', { status: 400 });
  }

  const result = streamText({
    model: 'anthropic/claude-sonnet-4.5',
    prompt,
  });

  return result.toUIMessageStreamResponse();
}
```

## Discover available models

You can fetch the list of available models programmatically:

```ts
import { gateway } from 'ai';

const { models } = await gateway.getAvailableModels();

models.forEach(model => {
  console.log(`${model.id}: ${model.name}`);
});
```
