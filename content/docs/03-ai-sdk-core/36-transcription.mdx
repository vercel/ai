# Transcription

<Note type="warning">Transcription is an experimental feature.</Note>

The AI SDK provides the [`transcibe`](/docs/reference/ai-sdk-core/transcribe)
function to transcribe audio using a transcription model.

```ts
import { experimental_transcibe as transcibe } from 'ai';
import { openai } from '@ai-sdk/openai';

const { transcript } = await transcibe({
  model: openai.transcription('whisper-1'),
  audio: audioData,
});
```

The `audio` property can be a `Blob`, `File`, `Uint8Array`, `ArrayBuffer`, `Buffer`, `URL` or a string (e.g. a URL or base64 encoded data).

To access the generated transcript:

```ts
const text = transcript.text; // transcript text e.g. "Hello, world!"
const segments = transcript.segments; // array of segments with start and end times, if available
const language = transcript.language; // language of the transcript if available e.g. "en"
const duration = transcript.duration; // duration of the transcript if available in seconds
```

## Settings

### Provider-Specific settings

Transcription models often have provider or model-specific settings which you can set using the `providerOptions` parameter. The options for the provider (`openai` in the example below) become request body properties.

```ts highlight={"9"}
import { experimental_transcibe as transcibe } from 'ai';
import { openai } from '@ai-sdk/openai';

const { transcript } = await transcibe({
  model: openai.transcription('whisper-1'),
  audio: audioData,
  providerOptions: {
    openai: {
      timestampGranularities: ['word'],
    },
  },
});
```

### Abort Signals and Timeouts

`transcibe` accepts an optional `abortSignal` parameter of
type [`AbortSignal`](https://developer.mozilla.org/en-US/docs/Web/API/AbortSignal)
that you can use to abort the transcription process or set a timeout.

```ts highlight={"7"}
import { openai } from '@ai-sdk/openai';
import { experimental_transcibe as transcibe } from 'ai';

const { transcript } = await transcibe({
  model: openai.transcription('whisper-1'),
  audio: audioData,
  abortSignal: AbortSignal.timeout(1000), // Abort after 1 second
});
```

### Custom Headers

`transcibe` accepts an optional `headers` parameter of type `Record<string, string>`
that you can use to add custom headers to the transcription request.

```ts highlight={"7"}
import { openai } from '@ai-sdk/openai';
import { experimental_transcibe as transcibe } from 'ai';

const { transcript } = await transcibe({
  model: openai.transcription('whisper-1'),
  audio: audioData,
  headers: { 'X-Custom-Header': 'custom-value' },
});
```

### Warnings

Warnings (e.g. unsupported parameters) are available on the `warnings` property.

```ts
const { transcript, warnings } = await transcibe({
  model: openai.transcription('whisper-1'),
  audio: audioData,
});
```

### Error Handling

When `transcibe` cannot generate a valid transcript, it throws a [`AI_NoTranscriptGeneratedError`](/docs/reference/ai-sdk-errors/ai-no-transcript-generated-error).

This error can arise for any the following reasons:

- The model failed to generate a response
- The model generated a response that could not be parsed

The error preserves the following information to help you log the issue:

- `responses`: Metadata about the transcription model responses, including timestamp, model, and headers.
- `cause`: The cause of the error. You can use this for more detailed error handling.

```ts
import {
  experimental_transcibe as transcibe,
  NoTranscriptGeneratedError,
} from 'ai';

try {
  await transcibe({ model, audio });
} catch (error) {
  if (NoTranscriptGeneratedError.isInstance(error)) {
    console.log('NoTranscriptGeneratedError');
    console.log('Cause:', error.cause);
    console.log('Responses:', error.responses);
  }
}
```
