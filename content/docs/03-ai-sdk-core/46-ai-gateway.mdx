Vercel AI Gateway gives you access to models from OpenAI, Anthropic, Google, and other providers through a single interface. Instead of installing separate provider packages and managing multiple API keys, you configure one gateway and use any supported model with a string identifier.

This guide walks you through setting up AI Gateway and using it to call models from different providers.

## Why use AI Gateway

AI Gateway offers several benefits when building with the AI SDK:

- **Multi-provider access**: Call models from OpenAI, Anthropic, Google, Meta, xAI, Mistral, DeepSeek, and other providers without installing additional packages.
- **Unified authentication**: Use a single API key for all providers, or authenticate automatically with OIDC when deployed on Vercel.
- **Observability**: Monitor AI model usage, costs, and performance through the Vercel dashboard.
- **Provider flexibility**: Switch between models and providers by changing a string, with no code restructuring required.

## Prerequisites

- The AI SDK installed in your project (`npm install ai`).
- A Vercel account with an [AI Gateway API key](https://vercel.com/docs/ai-gateway).

## Steps

### 1. Set your API key

Create an AI Gateway API key from the [Vercel dashboard](https://vercel.com/docs/ai-gateway) and add it to your environment:

```bash filename=".env.local"
AI_GATEWAY_API_KEY=your_api_key_here
```

<Note>
When deployed to Vercel, the AI Gateway supports OIDC authentication, which handles credentials automatically without an API key. See the [AI Gateway authentication docs](/providers/ai-sdk-providers/ai-gateway#authentication) for details.
</Note>

### 2. Use a model string

The AI SDK routes requests through AI Gateway when you pass a model string in the `provider/model-name` format. No additional imports or provider packages are needed:

```ts
import { generateText } from 'ai';

const { text } = await generateText({
  model: 'anthropic/claude-sonnet-4',
  prompt: 'Explain how a neural network learns.',
});

console.log(text);
```

This works with any AI SDK function, including `streamText`, `generateText`, and agent workflows.

### 3. Stream text from a different provider

To switch providers, change the model string. The following example streams a response from an OpenAI model:

```ts
import { streamText } from 'ai';

const result = streamText({
  model: 'openai/gpt-4o',
  prompt: 'Write a short poem about distributed systems.',
});

for await (const chunk of result.textStream) {
  process.stdout.write(chunk);
}
```

### 4. Use the gateway provider instance (optional)

For cases where you need more control — such as wrapping the provider with [middleware](/docs/ai-sdk-core/middleware) or registering it in a [provider registry](/docs/ai-sdk-core/provider-management) — import the `gateway` provider instance:

```ts
import { generateText, gateway } from 'ai';

const { text } = await generateText({
  model: gateway('google/gemini-2.5-flash'),
  prompt: 'Summarize the benefits of serverless architecture.',
});

console.log(text);
```

You can also create a custom gateway instance to set a specific API key, base URL, or custom headers:

```ts
import { createGateway } from 'ai';

const myGateway = createGateway({
  apiKey: process.env.AI_GATEWAY_API_KEY ?? '',
});
```

### 5. Use AI Gateway in a Next.js API route

To build a deployed endpoint, create an API route that uses AI Gateway. The following example works with [Fluid compute](https://vercel.com/docs/fluid-compute) on Vercel for longer-running AI workloads:

```ts filename="app/api/chat/route.ts"
import { streamText } from 'ai';

export const maxDuration = 60;

export async function POST(request: Request) {
  const { prompt }: { prompt?: string } = await request.json();

  if (!prompt) {
    return new Response('Prompt is required', { status: 400 });
  }

  const result = streamText({
    model: 'anthropic/claude-sonnet-4',
    prompt,
  });

  return result.toUIMessageStreamResponse();
}
```

## Available models

AI Gateway supports models from OpenAI, Anthropic, Google, Meta, xAI, Mistral, DeepSeek, Amazon Bedrock, Cohere, Perplexity, and other providers.

You can discover available models programmatically:

```ts
import { gateway } from 'ai';

const availableModels = await gateway.getAvailableModels();

availableModels.models.forEach(model => {
  console.log(`${model.id}: ${model.name}`);
});
```

For the full list, see the [AI Gateway documentation](https://vercel.com/docs/ai-gateway).
