# Abort breaks resumable streams

## Issue

When using `useChat` with `resume: true`, built-in abort handling can break
stream resumption. Closing a tab, refreshing the page, or calling
`useChat().stop()` aborts the active HTTP request. If your server forwards
`req.signal` to `streamText`, the generation stops with that request, so the
resume endpoint has nothing left to reconnect to.

```tsx
// This setup will conflict with resumable streams
const { messages, stop } = useChat({
  id: chatId,
  resume: true,
});
```

## Why this happens

`stop()` and browser disconnects abort the underlying fetch request. With
resumable streams, the generation needs to keep running independently of that
request so the client can reconnect later. If generation is tied directly to
`req.signal`, aborting the request also aborts the generation.

## Solutions

### Option 1: Use built-in abort handling without stream resumption

If you want to use the built-in `stop()` helper, keep the default
request-bound abort behavior and disable resumable streams:

```tsx
const { messages, sendMessage, stop } = useChat({
  id: chatId,
  resume: false, // Disable stream resumption (default behavior)
});
```

### Option 2: Use stream resumption with a custom stop endpoint

If you need both resumable streams and a manual Stop action, move cancellation
state to the server:

1. Create a server-owned `AbortController` for each active generation
2. Store the controller (or a lookup key for it) alongside the `activeStreamId`
3. Accumulate the assistant response on the server while streaming
4. Add a `POST /api/chat/[id]/stop` endpoint that aborts the controller
5. Persist the partial assistant message and clear the `activeStreamId` when stopping

<Note>
  Call your stop endpoint from the UI instead of relying only on the built-in
  `stop()` helper. Treat the server's accumulated assistant response as the
  source of truth for persistence.
</Note>

This is a custom architecture that you implement on top of the SDK. The SDK
supports the primitives for it, but it does not create the stop endpoint or
server-side controller registry for you.
