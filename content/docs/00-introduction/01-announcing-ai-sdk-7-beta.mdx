# Announcing AI SDK 7 Beta

<Note type="warning">
  AI SDK 7 is in beta — while more stable than alpha, AI SDK 7 is still in
  active development and APIs may still change. Pin to specific versions as
  breaking changes may occur in patch releases.
</Note>

## Why AI SDK 7?

AI SDK 7 is a **major version** because the provider specification continues to evolve quickly. Like AI SDK 6, most application-level code should continue to work with limited changes, but there are a few important upgrades and breaking changes — most notably **ESM-only packages**, a new **telemetry architecture**, and several API cleanups across tools and providers.

If you're building with `generateText`, `streamText`, `ToolLoopAgent`, and `useChat`, the upgrade should generally be straightforward. For the full list of changes, see the [AI SDK 6.x to 7.0 migration guide](/docs/migration-guides/migration-guide-7-0).

## Beta Version Guidance

The AI SDK 7 Beta is intended for:

- **Trying new features** like `WorkflowAgent`, top-level reasoning, and provider-independent uploads
- **Validating existing AI SDK 6 apps** against the migration guide before stable release
- **Giving us feedback** on the APIs, docs, and migration experience

Your feedback during the beta helps shape the final stable release. Share it through [GitHub issues](https://github.com/vercel/ai/issues/new/choose).

## Installation

To install the AI SDK 7 Beta, run:

```bash
npm install ai@beta @ai-sdk/openai@beta @ai-sdk/react@beta
```

If you want to use telemetry or `WorkflowAgent`, install the optional packages as well:

```bash
npm install @ai-sdk/otel@beta @ai-sdk/workflow@beta workflow
```

## What's New in AI SDK 7?

AI SDK 7 Beta introduces several new capabilities (with more to come soon):

### WorkflowAgent

AI SDK 7 introduces `WorkflowAgent`, a durable agent implementation built on the [Workflow SDK](https://workflow-sdk.dev/). It turns your agent run into a durable workflow where each tool invocation becomes its own step with automatic retries. This means your agent is not bound by function duration limits, and if any single step fails, it picks up where it left off instead of restarting from scratch.

Unlike `DurableAgent` from earlier releases, `WorkflowAgent` accepts provider model instances directly (e.g. `openai('gpt-5.4')`), just like `ToolLoopAgent`.

```ts
import { WorkflowAgent } from '@ai-sdk/workflow';
import { tool } from 'ai';
__PROVIDER_IMPORT__;
import { z } from 'zod';

export const supportAgent = new WorkflowAgent({
  model: __MODEL__,
  instructions: 'You are a helpful support agent.',
  tools: {
    lookupOrder: tool({
      description: 'Look up an order by ID',
      inputSchema: z.object({
        orderId: z.string(),
      }),
      execute: async ({ orderId }) => {
        'use step';

        /* look up order in database */

        return {
          orderId,
          status: 'shipped',
        };
      },
    }),
  },
});
```

On the UI side, `WorkflowChatTransport` lets `useChat` reconnect and resume interrupted streams automatically. Learn more in the [WorkflowAgent documentation](/docs/agents/workflow-agent).

### Overhauled Telemetry

AI SDK 7 overhauls telemetry to make observability much easier to adopt. OpenTelemetry support now lives in the new `@ai-sdk/otel` package, and setup is simple: register a telemetry integration once at app startup, and all AI SDK calls are traced automatically.

```ts
import { registerTelemetry, streamText } from 'ai';
import { GenAIOpenTelemetry } from '@ai-sdk/otel';
__PROVIDER_IMPORT__;

registerTelemetry(new GenAIOpenTelemetry());

const result = streamText({
  model: __MODEL__,
  prompt: "Summarize today's release notes.",
});
```

Register an integration once globally or swap integrations per call. Use `GenAIOpenTelemetry` for semantic-convention spans or `OpenTelemetry` for legacy AI SDK spans. Send traces to providers like [Braintrust](/providers/observability/braintrust), [Langfuse](/providers/observability/langfuse), and [MLflow](/providers/observability/mlflow), or inspect them locally with `DevToolsTelemetry()`.

Observability integrations are already available across the ecosystem. Some integrate via `@ai-sdk/otel`; others, like [LangSmith](/providers/observability/langsmith), [Axiom](/providers/observability/axiom), and [Helicone](/providers/observability/helicone), connect through wrappers or provider-level instrumentation instead.

Learn more in the [Telemetry documentation](/docs/ai-sdk-core/telemetry), [Observability Integrations](/providers/observability), and [DevTools documentation](/docs/ai-sdk-core/devtools).

### Top-level Reasoning

Reasoning is now a top-level setting on `generateText` and `streamText`, so you can control thinking across providers with one portable API instead of reaching for provider-specific options first.

```ts
import { generateText } from 'ai';
__PROVIDER_IMPORT__;

const { text, reasoning } = await generateText({
  model: __MODEL__,
  reasoning: 'high',
  prompt: 'Explain quantum entanglement in simple terms.',
});
```

The same setting also works on `ToolLoopAgent`:

```ts
import { ToolLoopAgent } from 'ai';
__PROVIDER_IMPORT__;

const agent = new ToolLoopAgent({
  model: __MODEL__,
  reasoning: 'high',
  instructions: 'You are a helpful physics tutor.',
});

const result = await agent.stream({
  prompt: 'Explain quantum entanglement in simple terms.',
});

for await (const chunk of result.textStream) {
  process.stdout.write(chunk);
}
```

Learn more in the [Reasoning documentation](/docs/ai-sdk-core/reasoning).

### File and Skill Uploads

AI SDK 7 adds provider-independent upload APIs for files (`uploadFile`) and skills (`uploadSkill`). Both functions return provider references that you can reuse in later model calls and even merge across providers when needed.

#### File Uploads

```ts
import { uploadFile, generateText } from 'ai';
import { openai } from '@ai-sdk/openai';
import fs from 'node:fs';

const { providerReference } = await uploadFile({
  api: openai.files(),
  data: fs.readFileSync('./photo.png'),
  filename: 'photo.png',
});

const { text } = await generateText({
  model: openai.responses('gpt-5.4'),
  messages: [
    {
      role: 'user',
      content: [
        { type: 'text', text: 'Describe what you see in this image.' },
        { type: 'image', image: providerReference },
      ],
    },
  ],
});
```

#### Skill Uploads

```ts
import { uploadSkill } from 'ai';
import { anthropic } from '@ai-sdk/anthropic';
import { readFileSync } from 'node:fs';

const { providerReference } = await uploadSkill({
  api: anthropic.skills(),
  files: [
    {
      path: 'my-skill/SKILL.md',
      content: readFileSync('./SKILL.md'),
    },
  ],
  displayTitle: 'My Skill',
});
```

Learn more in the [File Uploads documentation](/docs/ai-sdk-core/file-uploads) and [Skill Uploads documentation](/docs/ai-sdk-core/skill-uploads).

### Stable Typed Tool Context

AI SDK 7 promotes `experimental_context` to stable, per-tool context via `toolsContext`. Each tool declares its own `contextSchema`, and context is passed per tool at the call site through the `toolsContext` map. This gives each tool only the context it declares, with full end-to-end type inference.

<Note>
  This API is still undergoing changes ([#14515](https://github.com/vercel/ai/pull/14515)). The example below reflects the current direction but may change before stable release.
</Note>

```ts
import { generateText, tool } from 'ai';
__PROVIDER_IMPORT__;
import { z } from 'zod';

const weather = tool({
  inputSchema: z.object({
    location: z.string(),
  }),
  contextSchema: z.object({
    weatherApiKey: z.string(),
  }),
  execute: async ({ location }, { context: { weatherApiKey } }) => {
    return getWeather(location, weatherApiKey);
  },
});

const result = await generateText({
  model: __MODEL__,
  tools: { weather },
  toolsContext: {
    weather: { weatherApiKey: process.env.WEATHER_API_KEY! },
  },
  prompt: 'What is the weather in San Francisco?',
});
```

This typed context also flows into approval callbacks and other tool lifecycle hooks. See the [migration guide](/docs/migration-guides/migration-guide-7-0) for upgrade details.

### Granular Timeout Controls

Timeouts now work at multiple levels: total call, individual step, stalled stream chunk, default tool execution, and per-tool overrides.

```ts
import { generateText } from 'ai';
__PROVIDER_IMPORT__;

const result = await generateText({
  model: __MODEL__,
  tools: {
    weather: weatherTool,
    slowApi: slowApiTool,
  },
  timeout: {
    totalMs: 60_000,
    stepMs: 10_000,
    toolMs: 5_000,
    tools: {
      slowApiMs: 15_000,
    },
  },
  prompt: 'What is the weather in San Francisco?',
});
```

Learn more in the [Settings documentation](/docs/ai-sdk-core/settings#timeout).

### Top-level `toolApproval`

AI SDK 7 adds a top-level `toolApproval` setting on `generateText`, `streamText`, and `ToolLoopAgent`. Instead of setting `needsApproval` on each tool individually, you can configure approval logic in one place:

```ts
import { generateText } from 'ai';
__PROVIDER_IMPORT__;

const result = await generateText({
  model: __MODEL__,
  tools: { weather: weatherTool, payment: paymentTool },
  toolApproval: {
    // Require approval for specific tools
    payment: true,
    // Or use a function for dynamic approval
    weather: async ({ args }) => args.city === 'restricted-zone',
  },
  prompt: 'Process payment and check weather.',
});
```

### `reasoning-file` Content Part

Models that return files as part of their reasoning trace (separate from regular output files) now use a distinct `reasoning-file` content part type. Previously these were grouped under the generic `file` type. If you have exhaustive part handling (`switch` statements, renderers, serializers), you will need to add a case for `type: 'reasoning-file'`.

## Migration from AI SDK 6.x

AI SDK 7 should be a relatively small migration for most applications, but there are a few high-impact changes to review:

- **All packages are now ESM-only** — `require()` is no longer supported
- **Telemetry moved out of `ai`** into the new `@ai-sdk/otel` package
- **`experimental_context` was renamed to `context`** and is now typed
- **Common call settings were clarified** across language model and request options

If you use telemetry, register the new telemetry integration once at startup:

```ts
import { registerTelemetry } from 'ai';
import { OpenTelemetry } from '@ai-sdk/otel';

registerTelemetry(new OpenTelemetry());
```

Start with the [AI SDK 6.x to 7.0 migration guide](/docs/migration-guides/migration-guide-7-0) and test against pinned beta versions while APIs continue to settle.
