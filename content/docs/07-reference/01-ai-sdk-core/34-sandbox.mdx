# `Sandbox`

The `Sandbox` interface describes an execution environment that tools can use to
run commands and read files. Pass a sandbox to `generateText`, `streamText`,
`ToolLoopAgent.generate`, `ToolLoopAgent.stream`, or agent UI stream helpers to
make it available to tool execution.

Passing a sandbox does not sandbox the tool itself. Tool code still runs in your
application process unless the tool explicitly delegates work to the sandbox.

## Type Definition

```ts
type Sandbox = {
  readonly description: string;
  readonly executeCommand: (options: { command: string }) => PromiseLike<{
    exitCode: number;
    stdout: string;
    stderr: string;
  }>;
  readonly readFile: (options: { path: string }) => PromiseLike<{
    content: string;
  }>;
};
```

## Example

```ts
const result = await generateText({
  model: __MODEL__,
  tools: { shell, readFile },
  sandbox,
  prompt: 'Read package.json and run the test suite.',
});
```

Inside a tool, read the sandbox from the second `execute` argument:

```ts
const shell = tool({
  inputSchema: z.object({ command: z.string() }),
  execute: async ({ command }, { sandbox }) => {
    if (!sandbox) {
      throw new Error('Sandbox is not available');
    }

    return sandbox.executeCommand({ command });
  },
});

const readFile = tool({
  inputSchema: z.object({ path: z.string() }),
  execute: async ({ path }, { sandbox }) => {
    if (!sandbox) {
      throw new Error('Sandbox is not available');
    }

    return (await sandbox.readFile({ path })).content;
  },
});
```
