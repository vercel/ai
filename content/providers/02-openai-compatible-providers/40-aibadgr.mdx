# AI Badgr Provider

[AI Badgr](https://aibadgr.com/) provides access to AI models through an OpenAI-compatible API that you can use with the AI SDK. AI Badgr is well-suited for high-volume, cost-sensitive workloads.

## Setup

The AI Badgr provider is available via the `@ai-sdk/openai-compatible` module as it is compatible with the OpenAI API.
You can install it with:

<Tabs items={['pnpm', 'npm', 'yarn', 'bun']}>

## Provider Instance

To use AI Badgr, you can create a custom provider instance with the `createOpenAICompatible` function from `@ai-sdk/openai-compatible`:

```ts
import { createOpenAICompatible } from '@ai-sdk/openai-compatible';

const aibadgr = createOpenAICompatible({
  name: 'aibadgr',
  baseURL: process.env.AIBADGR_BASE_URL ?? 'https://aibadgr.com/api/v1',
  headers: {
    Authorization: `Bearer ${process.env.AIBADGR_API_KEY}`,
  },
});
```

<Note>
  You can obtain an API key from the AI Badgr dashboard. Optionally set
  `AIBADGR_BASE_URL` to override the default base URL.
</Note>

## Language Models

You can interact with AI Badgr models using a provider instance. AI Badgr uses a tier-based system for easy model selection:

```ts
const model = aibadgr('premium');
```

### Recommended Models (Tiers)

AI Badgr provides tier-based model access:

- **`basic`** - Fast and cost-effective for simple tasks
- **`normal`** - Balanced performance and cost for general use
- **`premium`** - Best quality for complex reasoning and generation

### Advanced: Pin Underlying Models

You can also specify exact model names if needed:

```ts
const model = aibadgr('gpt-4o');
const model2 = aibadgr('claude-3-5-sonnet-20241022');
```

### Example - Generate Text

You can use AI Badgr language models to generate text with the `generateText` function:

```ts
import { createOpenAICompatible } from '@ai-sdk/openai-compatible';
import { generateText } from 'ai';

const aibadgr = createOpenAICompatible({
  name: 'aibadgr',
  baseURL: process.env.AIBADGR_BASE_URL ?? 'https://aibadgr.com/api/v1',
  headers: {
    Authorization: `Bearer ${process.env.AIBADGR_API_KEY}`,
  },
});

const { text, usage, finishReason } = await generateText({
  model: aibadgr('premium'),
  prompt: 'Write a vegetarian lasagna recipe for 4 people.',
});

console.log(text);
console.log('Token usage:', usage);
console.log('Finish reason:', finishReason);
```

### Example - Stream Text

AI Badgr language models can also generate text in a streaming fashion with the `streamText` function:

```ts
import { createOpenAICompatible } from '@ai-sdk/openai-compatible';
import { streamText } from 'ai';

const aibadgr = createOpenAICompatible({
  name: 'aibadgr',
  baseURL: process.env.AIBADGR_BASE_URL ?? 'https://aibadgr.com/api/v1',
  headers: {
    Authorization: `Bearer ${process.env.AIBADGR_API_KEY}`,
  },
});

const result = streamText({
  model: aibadgr('premium'),
  prompt: 'Write a short story about AI.',
});

for await (const textPart of result.textStream) {
  process.stdout.write(textPart);
}

console.log();
console.log('Token usage:', await result.usage);
console.log('Finish reason:', await result.finishReason);
```

AI Badgr language models can also be used with other AI SDK functions like `generateObject` and `streamObject`.

<Note>
  Model support for tool calls and structured object generation may vary. Check
  the AI Badgr documentation for specific supported features for each model.
</Note>
