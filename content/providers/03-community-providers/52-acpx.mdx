# acpx

[`acpx-ai-provider`](https://www.npmjs.com/package/acpx-ai-provider) brings the major coding agents â€” **Claude Code, Codex CLI, Gemini CLI, GitHub Copilot, Cursor, Pi**, and any other adapter that speaks the [Agent Client Protocol](https://agentclientprotocol.com/) â€” to the AI SDK as first-class language models. `generateText`, `streamText`, `generateObject`, persistent sessions, MCP tools: all of it works the same way you already use the SDK with any other provider.

```typescript
import { createAcpxProvider } from 'acpx-ai-provider';
import { generateText } from 'ai';

const provider = createAcpxProvider({ agent: 'claude' });

const { text } = await generateText({
  model: provider.languageModel(),
  prompt: 'List the files in this repo and summarise what each does.',
});
```

That's the whole setup. No agent CLI to install separately. No `{ command, args }` spawn config to hand-roll per agent. The underlying [`acpx`](https://www.npmjs.com/package/acpx) runtime ships an agent registry, resolves the adapter on first use, `npx`-spawns it, persists the session on disk, normalises the event stream, and handles authentication through env vars (or the host CLI's existing credential store for adapters like Copilot).

## Built-in agents

`acpx` ships registry entries for the major coding agents out of the box. Pick one with `agent:`:

| `agent:` value | Backing adapter         |
| -------------- | ----------------------- |
| `claude`       | Claude Code (Anthropic) |
| `codex`        | Codex CLI (OpenAI)      |
| `gemini`       | Gemini CLI (Google)     |
| `copilot`      | GitHub Copilot CLI      |
| `cursor`       | Cursor agent            |
| `pi`           | Pi adapter              |

Need a custom adapter? Pass `agentRegistryOverrides` and `acpx` will use yours instead. See the [`acpx` package docs](https://www.npmjs.com/package/acpx) for the canonical list and the override shape.

<Note type="warning">
  This package is alpha (v0.x). The public surface is stable enough to use
  today, but expect changes between minor versions. See
  [Limitations](#limitations) before adopting in production.
</Note>

## Version compatibility

| Provider version | AI SDK version | npm tag  | Status |
| ---------------- | -------------- | -------- | ------ |
| 0.x              | v6             | `latest` | Alpha  |

The provider implements `LanguageModelV2` and runs under AI SDK v6 in compatibility mode. `LanguageModelV3` migration tracks runtime feature work upstream.

`acpx` is the runtime that resolves and spawns ACP adapters. `acpx-ai-provider` is the thin layer that turns it into a `LanguageModelV2`. You don't interact with `acpx` directly â€” it's a peer dependency so the SDK and the provider agree on a single runtime version.

## Provider Instance

```typescript
import { createAcpxProvider } from 'acpx-ai-provider';

const provider = createAcpxProvider({
  agent: 'claude', // any built-in id, see "Built-in agents"
  cwd: process.cwd(),
});
```

The first call against any agent lazily `npx`-spawns the adapter. Subsequent calls against the same `cwd` reuse the cached binary.

## Examples

### `streamText`

```typescript
import { createAcpxProvider } from 'acpx-ai-provider';
import { streamText } from 'ai';

const provider = createAcpxProvider({ agent: 'codex' });

const result = streamText({
  model: provider.languageModel(),
  prompt: 'Walk me through the files in this directory.',
});

for await (const chunk of result.textStream) {
  process.stdout.write(chunk);
}
```

### `generateObject`

JSON mode works through the runtime's structured-output transform, which strips markdown fences so the SDK's parser sees clean JSON:

```typescript
import { createAcpxProvider } from 'acpx-ai-provider';
import { generateObject } from 'ai';
import { z } from 'zod';

const provider = createAcpxProvider({ agent: 'claude' });

const { object } = await generateObject({
  model: provider.languageModel(),
  schema: z.object({
    name: z.string(),
    ingredients: z.array(z.string()),
  }),
  prompt: 'Give me a recipe for chocolate chip cookies.',
});
```

### Persistent multi-turn sessions

The same agent process can stay warm across turns. Pin it with a `sessionKey`:

```typescript
const provider = createAcpxProvider({
  agent: 'claude',
  sessionMode: 'persistent',
  sessionKey: 'my-app::project-x',
});

await generateText({
  model: provider.languageModel(),
  prompt: 'List my open files.',
});
await generateText({
  model: provider.languageModel(),
  prompt: 'Summarise the largest one.',
});
```

The ACP session is persisted on disk, so it survives process restarts too â€” pass the same `sessionKey` later and the conversation resumes.

### MCP servers

Pass MCP server configs through to the agent, useful for adding tools the agent doesn't bundle:

```typescript
const provider = createAcpxProvider({
  agent: 'claude',
  mcpServers: [
    {
      type: 'stdio',
      name: 'fs',
      command: 'npx',
      args: ['-y', '@modelcontextprotocol/server-filesystem', '/tmp'],
    },
  ],
});
```

## Authentication

Auth is env-var or config-file driven. Each adapter reads its own credentials â€” typically:

```bash
ACPX_AUTH_ANTHROPIC_API_KEY=sk-ant-...
ACPX_AUTH_OPENAI_API_KEY=sk-...
ACPX_AUTH_GOOGLE_API_KEY=...
```

For adapters that use a host CLI login (e.g. `gh auth login` for Copilot), run that login flow first; `acpx` reuses the existing credential store.

## Limitations

- **Alpha.** v0.x â€” public API may shift between minor versions.
- **Tools via MCP only.** Host-side AI SDK tools (the `acpTools()` / TCP-callback pattern from other ACP integrations) are not implemented.
- **`usage_update` is last-write-wins.** Only the most recent `usage_update` from the runtime survives onto the `finish` part.
- **Permission policy is mode-based by default.** Pick `approve-all`, `approve-reads`, or `deny-all` up front; a per-call callback hook is in flight.
- **`npx` cold start on first use** of any agent in a given `cwd`. Subsequent calls in the same `cwd` reuse the cached binary.
- **`LanguageModelV2` in v6 compat mode.** V3 migration tracks runtime feature work.

The full limitations list and per-agent quirks live in the [package README](https://github.com/DaniAkash/acpx/tree/main/packages/acpx-ai-provider#known-limitations).
