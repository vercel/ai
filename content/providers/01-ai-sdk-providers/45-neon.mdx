# Neon Provider

The [Neon](https://neon.com) provider gives the AI SDK access to the **Neon AI Gateway**, a branch-scoped gateway that serves a unified catalog of foundation models (Anthropic, OpenAI, Google, Meta, and more) through an OpenAI-compatible endpoint.

Because the gateway is branch-scoped, both the gateway URL and the platform token come from the Neon Console: open your project, select a branch, and go to the **AI Gateway** tab. The token is shown only once when you generate it.

## Setup

The Neon provider is available via the `@ai-sdk/neon` module. You can install it with:

## Configuration

The gateway URL is branch-scoped and therefore has no default — you must provide it. Set both values from the **AI Gateway** tab as environment variables:

```bash
# The branch host root; the provider appends the gateway path internally.
NEON_AI_GATEWAY_BASE_URL="https://<branch-id>-api.ai.<region>.aws.neon.tech"
# The platform token (shown once when generated).
NEON_AI_GATEWAY_TOKEN="nt_live_..."
```

## Provider Instance

You can import the default provider instance `neon` from `@ai-sdk/neon`:

```ts
import { neon } from '@ai-sdk/neon';
```

For custom configuration, you can import `createNeon` and create a provider instance with your settings:

```ts
import { createNeon } from '@ai-sdk/neon';

const neon = createNeon({
  baseURL: process.env.NEON_AI_GATEWAY_BASE_URL,
  apiKey: process.env.NEON_AI_GATEWAY_TOKEN,
});
```

You can use the following optional settings to customize the Neon provider instance:

- **baseURL** _string_

  The branch-scoped gateway host root, e.g. `https://<branch-id>-api.ai.<region>.aws.neon.tech`. The unified gateway path is appended automatically. Defaults to the `NEON_AI_GATEWAY_BASE_URL` environment variable.

- **apiKey** _string_

  The platform token sent using the `Authorization` header. Defaults to the `NEON_AI_GATEWAY_TOKEN` environment variable.

- **headers** _Record&lt;string,string&gt;_

  Custom headers to include in the requests.

- **fetch** _(input: RequestInfo, init?: RequestInit) => Promise&lt;Response&gt;_

  Custom [fetch](https://developer.mozilla.org/en-US/docs/Web/API/fetch) implementation.

## Language Models

You can create language models using a provider instance:

```ts
import { neon } from '@ai-sdk/neon';
import { generateText } from 'ai';

const { text } = await generateText({
  model: neon('databricks-claude-haiku-4-5'),
  prompt: 'Summarize Postgres for me.',
});
```

Neon language models can also be used in the `streamText` function
(see [AI SDK Core](/docs/ai-sdk-core)).

You can also use the `.languageModel()` and `.chat()` methods:

```ts
const model = neon.languageModel('databricks-claude-haiku-4-5');
const model = neon.chat('databricks-claude-haiku-4-5');
```

## Routing

The Neon gateway exposes provider-native endpoints alongside a unified, OpenAI-compatible MLflow endpoint. To maximize feature coverage — and to support models that are only served natively, such as **Codex** — the provider routes each model to the best endpoint based on its id. This is transparent: you always use the same `neon('databricks-...')` call, base URL, and token.

| Model family                          | Endpoint              | Notes                                                              |
| ------------------------------------- | --------------------- | ----------------------------------------------------------------- |
| Anthropic (`databricks-claude-*`)     | native Messages API   | Streaming structured output and native reasoning.                 |
| OpenAI (`databricks-gpt-*`, `*-codex`)| native Responses API  | Required for Codex; native reasoning and the image-generation tool.|
| Everything else (Gemini, Llama, ...)  | unified MLflow endpoint| Broad coverage. Gemini is routed here because its native gateway endpoint does not support streaming. |

## Capabilities and Limitations

Verified across Anthropic, OpenAI (including Codex), Google, and Meta models:

- `generateText` / `streamText` (text generation, system prompts, multi-turn)
- Tool calling (`generateText` / `streamText`, single and multi-step)
- `generateObject` / `streamObject` (structured output)
- Image (vision) input

### Sampling parameters

Native-routed models (Anthropic, OpenAI) use the official provider's parameter handling. For MLflow-routed models, the gateway proxies to heterogeneous backends that accept different parameter subsets, so the provider detects the model family and **drops parameters a backend doesn't support, emitting an AI SDK warning** (`result.warnings`) instead of failing the request. Detection is heuristic (prefix-based); unknown models are passed through unchanged.

| Parameter                              | Dropped (with a warning) for       |
| -------------------------------------- | ---------------------------------- |
| `frequencyPenalty` / `presencePenalty` | Meta (Llama) models                |
| `seed`                                 | Meta (Llama) models                |
| `reasoningEffort` (provider option)    | Gemini models                      |

### Image generation

Image generation is available on OpenAI models via the Responses `image_generation` tool (there is no `generateImage()` image-model endpoint). Use `streamText` rather than `generateText`: streaming returns the image as a `tool-result` part and avoids the gateway's non-streaming response-size cap and read timeout (a single blocking call returns a large base64 body that can exceed those limits).

```ts
import { streamText } from 'ai';
import { neon } from '@ai-sdk/neon';
import { imageGeneration } from '@ai-sdk/openai/internal';

const result = streamText({
  model: neon('databricks-gpt-5-mini'),
  prompt: 'Generate an image of a red apple on a wooden table',
  tools: { image: imageGeneration({ partialImages: 3 }) },
});

for await (const part of result.fullStream) {
  if (part.type === 'tool-result' && 'result' in part.output) {
    const png = Buffer.from(part.output.result as string, 'base64');
    // save or use the image
  }
}
```

### Not offered by the gateway

- **`generateImage()`** and **embeddings (`embed` / `embedMany`)** throw `NoSuchModelError` — there are no image-model or embeddings endpoints on the gateway.
- **`gpt-oss-*`** models return a non-standard ("harmony") response shape on the unified endpoint and are not fully supported.
